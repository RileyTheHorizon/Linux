# 32. Статическая и динамическая маршрутизация, OSPF

## Задание
1. Развернуть 3 виртуальные машины
2. Объединить их разными vlan
- настроить OSPF между машинами на базе Quagga;
- изобразить ассиметричный роутинг;
- сделать один из линков "дорогим", но что бы при этом роутинг был симметричным.


1. Асимметричный роутинг
Трафик между R1 и R2 идет через R3, а не напрямую

Достигнуто через манипуляцию OSPF cost:

R1-R2: cost 1000 (дорогой)

R1-R3: cost 10 (дешевый)

R2-R3: cost 10 (дешевый)

2. Симметричность "дорогого" линка
Линк R1-R2 одинаково "дорогой" в обе стороны

Обеспечивает предсказуемость маршрутизации

3. Технические детали
FRR вместо Quagga (более современное решение)
